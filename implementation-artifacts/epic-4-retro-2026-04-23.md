## New Finding: S04.02 Critical Review-Fix (2026-04-23)

During the 2026-04-23 review pass, a **critical regression** was identified in
`services/ai-gateway/src/ai_gateway/kraftdata_client.py`:

- **Bug:** `httpx.AsyncClient` was configured with `X-API-Key: {api_key}` as the
  default header instead of the correct `Authorization: Bearer {api_key}`. The
  `Content-Type: application/json` header was also missing.
- **Root cause:** An earlier scope-cleanup pass (removing circuit breaker / retry
  from `call_kraftdata`) inadvertently replaced the correct Auth header with the
  wrong format.
- **Fix applied:** Authorization header restored to `Authorization: Bearer {api_key}`;
  `Content-Type: application/json` added to default headers.
- **Impact:** All KraftData API calls would have returned 401 in production. The
  functional integration tests (S04.10) used a respx mock that did not enforce auth
  header format — so this bug survived the full integration test suite.

```
[ANTI-PATTERN] Integration test mocks did not enforce auth header contract
IMPACT: prompt_adjustment
SEVERITY: high
```

```
[ACTION] Add a respx mock assertion that verifies `Authorization: Bearer` header format on all KraftData outbound requests
IMPACT: prompt_adjustment
SEVERITY: high
```

```
[PATTERN] Scope-cleanup PRs must include header contract regression tests
IMPACT: standards_update
SEVERITY: medium
```

When refactoring responsibility boundaries (e.g., moving resilience logic out of the
HTTP client), all integration tests must explicitly assert that the HTTP headers
configured on the client are correct. A mock that accepts any Authorization format is
insufficient — it must validate the exact header name and value pattern used by the
external provider.

## What Went Well

### [PATTERN] All Three High-Priority Risks Fully Mitigated
`IMPACT: standards_update | SEVERITY: medium`

E04-R-001 (webhook HMAC timing attack), E04-R-002 (SSE stream fragility), E04-R-003
(Redis XADD publish gap) — all three Score-6 risks were fully addressed:

- **HMAC:** `hmac.compare_digest()` with unit test asserting source-code usage + timing assertion < 1ms.
- **SSE:** Queue-based async producer/consumer (upstream_reader + heartbeat_sender), partial frame buffering, `asyncio.CancelledError` propagation cancels both tasks on disconnect.
- **Redis publish gap:** try/except on XADD, ERROR-level structured log with `execution_id`, `webhook_log.processed=False` on failure; still returns 200 to KraftData to prevent duplicate retries.

### [PATTERN] Two-Layer Resilience Composition
`IMPACT: standards_update | SEVERITY: low`

`circuit.call(lambda: with_retry(factory, agent_name))` is the established pattern.
Circuit breaker (outer) = CLOSED/OPEN/HALF_OPEN state machine with asyncio.Lock.
Retry (inner) = exponential backoff 1s→2s→4s, ±25% jitter, max 3 retries. Non-retryable
classes (CircuitOpenError, 4xx) propagate immediately through both layers. Reference
implementation: `services/ai-gateway/src/ai_gateway/kraftdata_resilient.py`.

### [PATTERN] YAML-Driven Agent Registry
`IMPACT: standards_update | SEVERITY: low`

`config/agents.yaml` maps 29 logical names → KraftData UUIDs/types/limits. `_NoDuplicatesLoader`
catches duplicate keys at load time. Hot-reload via `POST /admin/registry/reload` uses
`asyncio.Lock` to protect concurrent `resolve()` reads. KraftData UUID rotation = config
change + hot reload — no code deploy.

### [PATTERN] testcontainers + respx Integration Test Stack
`IMPACT: standards_update | SEVERITY: low`

S04.10 proves: session-scoped testcontainers PostgreSQL 16 + Redis 7, `respx.MockRouter`
for outbound KraftData calls, `httpx.AsyncClient(transport=ASGITransport(app, lifespan="on"))`
for full in-process FastAPI. 10 scenarios, 130 tests, ~10s runtime, 90%+ coverage, CI-safe.
This stack is mandatory for all backend integration test stories.

### [PATTERN] Fire-and-Forget Async DB Writes
`IMPACT: standards_update | SEVERITY: low`

`asyncio.create_task(_write_to_db(...))` for execution logs and webhook logs. DB failure
→ ERROR-level structured log (with `execution_id`) → never propagates 500 to caller.
Gateway remains operational during DB outages. Apply to all audit/logging writes.

### [PATTERN] Admin Introspection Endpoints
`IMPACT: standards_update | SEVERITY: low`

`GET /admin/circuits` (per-agent state), `GET /admin/rate-limit` (active/queued/rejected),
`GET /admin/executions` (filtered log, 5 filters, pagination), `POST /admin/registry/reload`.
All ClusterIP-only (no Ingress). Pattern: every stateful backend service must expose
comparable introspection endpoints as a standard acceptance criterion.

### [PATTERN] TRACE_GATE PASS on First Run
`IMPACT: standards_update | SEVERITY: low`

91.5% overall FULL coverage, 100% P0 coverage. All 59 acceptance criteria map to at
least one test (100% any coverage). Contrast: E03 required a Run 2 to close a P0 gap.

## Process Learnings

### [PROCESS_CHANGE] testcontainers + respx Is the Mandatory Backend Integration Test Stack
All backend epics must produce an S0X.10-equivalent story using testcontainers + respx.
No backend service can be marked `done` without this suite.

### [PROCESS_CHANGE] Two-Layer Resilience Pattern Is the Standard for Outbound HTTP
Circuit breaker (outer) wrapping retry (inner) is the established standard for E05+.
E05 calls scrapers; E08 calls Stripe; E09 calls email providers. Each must apply this
pattern — ad-hoc retry without circuit breaking is not acceptable.

### [PROCESS_CHANGE] ATDD Checklist Is a Before-Dev Gate
ATDD checklists must be scaffolded (with RED-phase assertions) before any development
story begins. E04 demonstrates the cost of skipping this: the test design is good but
there is no formal evidence of TDD discipline in the per-story record.

### [PROCESS_CHANGE] Admin Introspection Endpoints Are Mandatory for Stateful Services
Any story implementing a circuit breaker, rate limiter, connection pool, or async queue
must include an admin introspection endpoint as a standard acceptance criterion. Not
optional.

### [PROCESS_CHANGE] Auth Header Contract Must Be Asserted in Integration Mocks
When mocking outbound HTTP calls with respx, always assert the complete Authorization
header format — not just that a request was made. KraftData, Stripe, SendGrid, Google
— all have specific auth schemes. The mock must fail if the wrong scheme is used.
