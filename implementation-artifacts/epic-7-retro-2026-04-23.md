## Executive Summary

Epic 7 delivered EU Solicit's **flagship capability**: an AI-powered proposal workspace encompassing Tiptap rich-text editing, section-by-section SSE streaming from the AI Gateway, requirement checklists, compliance checking, clause risk analysis, scoring simulation, pricing recommendations, win theme extraction, content blocks library, PDF/DOCX export, and version history with diffs and rollback. It is the largest and most architecturally complex epic delivered (17 stories, 161 ACs), building on the AI Gateway (E04) and Opportunity Discovery (E06) foundations.

**All 17 stories are marked DONE as of 2026-04-23.** This retrospective covers the completed epic with full story closure.

**Headline Results:**
- **TRACE_GATE:** FAIL at mid-sprint gate (27.3% GREEN) — structural 17-story epic scale issue, not test quality failure. All stories subsequently completed.
- **NFR:** FAIL → RESOLVED — two P0 blockers (NFR4: raw SSE error exposure; NFR12: missing audit trail on `/generate`) detected by NFR assessment, remediated via injected hardening story S07.17.
- **ATDD:** 17/17 stories with ATDD checklists (~1,145 tests) — 2nd consecutive 100% checklist coverage epic.
- **TEA Review:** 1/17 scored (S07.11: 95/100) — 5th consecutive epic with inadequate TEA coverage.
- **R-001 (Data Loss, Score 9):** S07.04 optimistic locking implementation fully complete with concurrent write race test (`asyncio.gather` with testcontainers PostgreSQL).
- **5 critical carry-forwards** to Epic 8: k6 baseline (5th deferral), Dependabot (5th deferral), OBS-001 circuit breaker fix, `_RUN_ID_REGISTRY` Celery bug, openapi-typescript codegen gate.

### [PATTERN] Content-Hash Optimistic Locking Closes R-001 (Data Loss, Score 9)
`IMPACT: standards_update` `SEVERITY: high`

S07.04 implements the complete pattern: `SELECT FOR UPDATE` on `proposal_versions`, Python `hashlib.md5(json.dumps(content, sort_keys=True))` hash comparison (deterministic), `409 Conflict` with `{"error": "version_conflict", "latest_content": <current>}`, concurrent race test via `asyncio.gather` with testcontainers PostgreSQL. The `PATCH /content/sections/{key}` + `PUT /content` pair covers both auto-save and full-save paths. This is the reference implementation for all collaborative document editing in the platform.

### [PATTERN] Vitest Source-Inspection ATDD at Scale — S07.15 Reference (87 tests, all GREEN)
`IMPACT: standards_update` `SEVERITY: medium`

Frontend stories S07.11–S07.16 use Vitest static source-inspection (regex/import assertions, no runtime rendering) for structural ACs. S07.15 (87 tests, all GREEN) is the reference implementation. Runtime RTL/JSDOM reserved for user-interaction flows. This pattern produces 100+ stable CI tests per story without `act()` complexity.

### [PATTERN] ThreadPoolExecutor Mandatory for CPU-Bound FastAPI Endpoints
`IMPACT: standards_update` `SEVERITY: medium`

S07.10 (PDF/DOCX export) uses `asyncio.get_event_loop().run_in_executor(None, render_fn)` to offload WeasyPrint/python-docx rendering. Event loop blocking from synchronous rendering libraries is invisible in unit tests and only manifests under load. ATDD checklists for S07.10 include structural assertion that `run_in_executor` is used.

### [PATTERN] Reset-Stuck Background Task for Long-Running Async State Machines
`IMPACT: standards_update` `SEVERITY: medium`

S07.05 introduces `reset_stuck_proposals_task` (Celery Beat) to mark proposals stuck in `GENERATING` as `FAILED` after configurable TTL. Required AC for any story that introduces an in-progress status variant.

### [PATTERN] TEA Test Review Catches Architecture Contradictions Invisible to ATDD
`IMPACT: standards_update` `SEVERITY: high`

S07.11 TEA review (95/100) caught: (1) `useBreakpoint` threshold 1280px vs AC5 requirement of 1024px; (2) frontend `ProposalResponse` missing `current_version_number` and `generation_status`. Both invisible at implementation time, only surfaced via adversarial test review. TEA review is not optional.

## What Needs Improvement

### [ANTI-PATTERN] TRACE_GATE FAIL — Gate Design Incompatible with Large-Epic TDD
`[ACTION]` `IMPACT: standards_update` `SEVERITY: critical`

The 80% overall passing threshold penalises epic scope rather than test quality. 17-story epics in RED phase with all tests written correctly still fail the gate. The correct design: **per-completed-story GREEN gate** (each `done` story must have ≥1 GREEN test per P0 AC) + **overall coverage gate** as informational metric during sprint. Single threshold fails TDD methodology at scale.
