### What Was Delivered

Epic 6 implements the complete opportunity discovery and intelligence surface — the primary revenue-enforcement layer of EU Solicit where all four subscription tiers are gated:

- **S06.01** — `GET /api/v1/opportunities/search`: PostgreSQL FTS (`websearch_to_tsquery`) across title/description/authority, 6-filter composition (CPV, region, budget, deadline, status, type), cursor-based pagination with total_count stability
- **S06.02** — Tier-gated response serialization: `OpportunityFreeResponse` (6 fields) vs `OpportunityFullResponse` (all fields); TierGate as FastAPI per-route dependency; strict Pydantic field enforcement (`test_free_response_has_exactly_six_model_fields`)
- **S06.03** — Usage metering middleware: atomic Lua `GET + INCR + EXPIRE` script (`_USAGE_LUA`); testcontainers Redis race test; `X-Usage-Remaining` header; `fakeredis[lua]>=2.21` (lupa) required for EVAL
- **S06.04** — Opportunity listing API: browse mode (no FTS), sort-by (relevance/deadline/budget/published), `OpportunityFreeResponse[]` for free tier
- **S06.05** — Opportunity detail API: dual-session service (pipeline read-only + client write), related opportunities (over-fetch 10 → scope-filter to 5), AI summary retrieval *(carry-forward: ready-for-dev at sprint close)*
- **S06.06** — Document upload API: S3 presigned PUT URL (15-min expiry), ClamAV async scan, `scan_status: pending` on confirm, `SELECT FOR UPDATE` on package size
- **S06.07** — Document download API: presigned GET URL (10-min expiry), `scan_status == clean` gate, audit log, ownership check
- **S06.08** — AI summary generation API: SSE streaming (`delta → metadata → done`), `asyncio.Semaphore` cap (E06-R-004), Lua-based UsageGate, `client.ai_summaries` persistence, 24h cache with staleness detection; 4 critical patches from senior dev review
- **S06.09** — Opportunity listing page: dual table/card view, `useSearchParams` URL-driven state, TanStack Query infinite scroll
- **S06.10** — Search/filter components: CPV autocomplete, region checkboxes, budget slider, deadline range, debounced API calls, active filter chips, URL-driven state with `stale-closure prevention via searchParamsRef`
- **S06.11** — Opportunity detail page: 5-tab layout (Overview, Documents, Requirements, AI Analysis, Submission Guide), deadline countdown, tab state in URL hash
- **S06.12** — Document upload component: drag-drop, `cancelledIdsRef` cancellation tracking, scan status polling *(in review at sprint close: 4 patches applied)*
- **S06.13** — AI summary panel: native `fetch` + `ReadableStream` (not `EventSource`), progressive rendering, usage counter; cross-story query key mismatch patch (S06.11 key `["opportunity", id]`)
- **S06.14** — Upgrade prompt modal: global Axios interceptor (`registerUpgradeInterceptor(show)`), tier comparison table, 403/429 triggered, never swallows errors

## What Went Well

### [PATTERN] 100% ATDD Checklist Coverage — First Epic to Achieve This
`IMPACT: standards_update | SEVERITY: high`

All 14 stories had ATDD checklists generated in RED phase before implementation. The TDD RED→GREEN discipline was followed across all 16 test suites (~1,209 tests). This closes the recurring ATDD deficit flagged in E04 and E05 retrospectives.

### [PATTERN] TRACE_GATE PASS on First Run (3rd Consecutive Epic)
`IMPACT: standards_update | SEVERITY: medium`

E04, E05, and E06 all pass TRACE_GATE on first run. The test design → ATDD → traceability gate sequence is a reliably repeatable process.

### [PATTERN] TierGate as FastAPI Per-Route Dependency — Revenue Enforcement Pattern
`IMPACT: standards_update | SEVERITY: critical`

`OpportunityTierGate` implemented as a per-route FastAPI `Depends()` (not middleware) means it cannot be accidentally omitted when adding new endpoints. The strict `OpportunityFreeResponse` 6-field-only Pydantic model (verified by `test_free_response_has_exactly_six_model_fields`) prevents accidental field leakage through model evolution. This is the definitive pattern for any tier-enforced feature.

### [PATTERN] Atomic Lua Script for Redis Metering (E06-R-002 Fully Mitigated)
`IMPACT: standards_update | SEVERITY: critical`

`_USAGE_LUA` module-level constant (GET + conditional INCR + EXPIRE in single script) correctly prevents billing race conditions. The testcontainers Redis concurrency test validates this under real concurrent load. Note: `fakeredis[lua]>=2.21` (lupa package) required for EVAL support in unit tests.

### [PATTERN] Negative ATDD Assertions Prevent Silent Architectural Mistakes in SSE
`IMPACT: prompt_adjustment | SEVERITY: high`

The S06.13 ATDD checklist explicitly asserts `new EventSource` is NOT used (ENOENT-class structural assertion). This prevented the silent GET-only failure that `EventSource` would cause on a POST endpoint. Negative structural assertions in ATDD checklists are a proven technique for preventing invisible architectural mistakes.

### [PATTERN] URL-Driven Filter State — No `useState` for Shareable State
`IMPACT: standards_update | SEVERITY: medium`

S06.10 filter sidebar uses `useSearchParams()` + `router.replace()` for all filter state, with stale-closure prevention via `searchParamsRef`. Enables shareable URLs and avoids state-on-navigation-loss. Zero `useState` for URL-representable state is the correct frontend pattern.

### [PATTERN] `'use client'` Headless Orchestration Components — Mount Once in Layout
`IMPACT: standards_update | SEVERITY: medium`

S06.14's `UpgradeInterceptor` is a headless component: `'use client'` directive before imports, renders `return null`, mounts exactly once in layout. The `registerUpgradeInterceptor(show)` receives `show` as parameter rather than calling `useUpgradePromptStore.getState()` directly — prevents circular dependency between `packages/ui` and `apps/client`. Always calls `return Promise.reject(error)` to preserve TanStack Query error states.

### [PATTERN] Dual-Session Service Functions for Cross-Schema Reads
`IMPACT: standards_update | SEVERITY: medium`

S06.05/S06.08 service functions use `get_pipeline_readonly_session` for all pipeline reads and `get_db_session` for `client.*` writes — explicitly separate `MetaData` instances per schema. This prevents cross-schema FK violations and is the correct model for any service spanning the pipeline/client schema boundary.

### [PATTERN] Senior Dev Review Catches SSE Generator Lifecycle Bugs
`IMPACT: prompt_adjustment | SEVERITY: critical`

S06.08 senior dev review identified 4 critical patches before shipping: (1) `asyncio.shield(semaphore.acquire())` causing permanent permit leaks on timeout; (2) usage quota incremented before opportunity existence check (wasting quota on 404); (3) `stream_agent` asynccontextmanager not closing inner generator; (4) no terminal SSE event when AI Gateway stream ended without `done`. These are class-of-bug that unit tests rarely catch — async generator lifecycle bugs require adversarial code review.

## What Needs Improvement

### [ANTI-PATTERN] S06.08 Required 4 Critical Patches — SSE Generator Lifecycle Omissions
`[ACTION] IMPACT: prompt_adjustment SEVERITY: critical`

Four production-class bugs reached senior dev review: semaphore permit leak on timeout, quota wasted on 404, generator resource not closed, missing terminal SSE event. Root cause: no story template checklist for SSE generator correctness.

→ **Action:** Inject SSE generator lifecycle checklist into every SSE story spec: (1) usage/quota check BEFORE `StreamingResponse` creation, (2) generator closed in `finally` block, (3) terminal event (`done` or `error`) guaranteed regardless of upstream error, (4) fresh session from `session_factory` inside generator body — no request-scoped session.

### [ANTI-PATTERN] S06.13 Cross-Story Query Key Mismatch — Silent Cache Miss
